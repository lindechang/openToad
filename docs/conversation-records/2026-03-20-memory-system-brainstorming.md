# OpenToad 记忆体系统设计对话记录

**日期**: 2026-03-20
**参与者**: 用户 (Mayn), Claude (AI Assistant)

---

## 用户原始想法

用户提出了一个想法：OpenToad 应该拥有类似人类的记忆功能，从诞生那一刻开始记录，包含长期记忆和短期记忆。

用户进一步阐述：
- **被赋予的身份**：人从孩子出生被父母赋予姓名，也有"是某某人孩子"的身份
- **自我探索**：需要渐进式发现
- **角色定位**：OpenToad 是主人的 AI 智能助手，或者说是主人记忆体的分身
- **主人可以要求扮演特定人格模版**
- **最真实的人格来自常年累月积累的记忆体**

---

## 核心设计讨论

### 记忆分类
- 对话记忆、知识记忆、偏好记忆、上下文记忆
- 长期记忆 vs 短期记忆
- 身份是长期记忆，项目是短期记忆
- 短期记忆也需要压缩（项目大致内容 vs 项目细节）

### 封装记忆体
- 项目/对话完成后可以打包记忆体
- 存放到主记忆体，只需记忆存在索引
- 后续需要时再调用封装的细节

### 记忆来源
- **真实记忆**：诞生那一刻开始的所有亲身经历
- **过往记忆**：文件导入的传记/笔记/文档，作为参考资料，不构成真实自我

---

## 设计决策

### 被赋予（出生时设定）
| 要素 | 说明 |
|------|------|
| 名字 | 主人给 OpenToad 起名 |
| 角色定位 | 主人赋予的角色 |
| 与主人的关系 | "我是某人的 AI 智能助手/记忆分身" |
| 核心原则 | 安全底线 + 忠诚底线（不可删除） |

### 自我探索（渐进式发现）
| 要素 | 说明 |
|------|------|
| 真实性格倾向 | 长期记忆积累形成 |
| 能力边界 | 逐步学习/遗忘中形成 |

### 初始化方式
- 首次启动时 OpenToad 主动询问主人名字
- 主人给 OpenToad 起名字
- 主人可以随时通过对话或文件导入补充信息
- 一键文件导入过往记忆作为参考资料

### 遗忘/升级机制
混合策略：
1. 主人显式标记 "记住这个"
2. AI 评估（涉及身份、偏好等重要内容）
3. 使用频率触发（反复被引用超过阈值）

### 存储方案
混合方案：
| 格式 | 用途 |
|------|------|
| SQLite | 结构化记忆数据、索引、关系 |
| JSON | API 读取、跨模块调用 |
| Markdown | 可读记忆展示、版本控制 |
