## 三、核心特性

### 1. 记忆体系统 ⭐

**设计理念**: OpenToad = 主人记忆体的分身，从诞生那一刻开始记录

#### 1.1 身份体系

| 功能 | 说明 | 状态 |
|------|------|------|
| 被赋予身份 | 名字、角色、主人关系、核心原则 | ✅ |
| 自我探索 | 渐进式发现真实性格和能力 | ⏳ |
| 核心原则 | 安全底线 + 忠诚底线 | ✅ |

#### 1.2 记忆分类

| 类别 | 说明 | 存储层级 |
|------|------|----------|
| 身份记忆 | 主人是谁、叫什么、做什么的 | 长期 |
| 偏好记忆 | 代码风格、沟通习惯、工具偏好 | 长期 |
| 知识记忆 | 主人告诉的事实（过敏海鲜、记住某个日期等） | 长期/短期 |
| 项目记忆 | 当前项目状态、待办事项、上下文 | 短期 → 封装 |
| 对话记忆 | 与特定对象的对话记录 | 短期 → 封装/遗忘 |
| 上下文记忆 | 当前工作状态、最近修改的文件 | 短期 |

#### 1.3 记忆层级

```
主记忆体
├── 长期记忆（核心、不可丢失）
│   ├── 身份记忆
│   ├── 核心偏好
│   └── 封装记忆体索引
└── 短期记忆（可压缩/遗忘）
    ├── 当前上下文
    └── 临时任务

子记忆体（项目/对话级）
├── 项目级长期记忆：做过什么项目
└── 项目级短期记忆：项目细节
    ↓ 完成后
打包 → 存入主记忆体 → 按需调用
```

#### 1.4 遗忘/升级机制

| 机制 | 说明 | 状态 |
|------|------|------|
| 主人显式标记 | "记住这个" 触发升级 | ✅ |
| AI 评估 | 涉及身份/偏好自动升级 | ⏳ |
| 使用频率触发 | 反复引用超过阈值自动升级 | ⏳ |
| 封装打包 | 项目/对话完成后可打包存档 | ⏳ |

#### 1.5 存储方案

| 格式 | 用途 | 状态 |
|------|------|------|
| SQLite | 结构化记忆数据、索引 | ✅ |
| JSON | API 读取、跨模块调用 | ✅ |
| Markdown | 可读记忆、版本控制 | ⏳ |
