## 三、核心特性

### 1. 记忆体系统 ⭐

**设计理念**: OpenToad = 主人记忆体的分身，从诞生那一刻开始记录

#### 1.1 身份体系

| 功能 | 说明 | 状态 |
|------|------|------|
| 被赋予身份 | 名字、角色、主人关系、核心原则 | ✅ |
| 自我探索 | 渐进式发现真实性格和能力 | ⏳ |
| 核心原则 | 安全底线 + 忠诚底线 | ✅ |

#### 1.2 记忆分类

| 类别 | 说明 | 存储层级 |
|------|------|----------|
| 身份记忆 | 主人是谁、叫什么、做什么的 | 长期 |
| 偏好记忆 | 代码风格、沟通习惯、工具偏好 | 长期 |
| 知识记忆 | 主人告诉的事实（过敏海鲜、记住某个日期等） | 长期/短期 |
| 项目记忆 | 当前项目状态、待办事项、上下文 | 短期 → 归档 |
| 对话记忆 | 与特定对象的对话记录 | 短期 → 归档/遗忘 |
| 上下文记忆 | 当前工作状态、最近修改的文件 | 短期 |

#### 1.3 记忆存储

```
单一记忆体
├── 长期记忆（核心、不可丢失）
│   ├── 身份记忆
│   ├── 核心偏好
│   └── 重要知识
└── 短期记忆（可压缩/遗忘）
    ├── 当前上下文
    └── 临时任务
```

#### 1.4 遗忘/升级机制

| 机制 | 说明 | 状态 |
|------|------|------|
| 主人显式标记 | "记住这个" 触发升级 | ✅ |
| AI 评估 | 涉及身份/偏好自动升级 | ⏳ |
| 使用频率触发 | 反复引用超过阈值自动升级 | ⏳ |
| 归档打包 | 项目/对话完成后可归档存档 | ⏳ |

#### 1.5 存储方案

| 格式 | 用途 | 状态 |
|------|------|------|
| SQLite | 结构化记忆数据、索引 | ✅ |
| JSON | API 读取、跨模块调用 | ✅ |
| Markdown | 可读记忆、版本控制 | ⏳ |

#### 1.6 数据表结构

```sql
-- 记忆体元数据表
CREATE TABLE memory_meta (
    memory_id TEXT PRIMARY KEY,      -- UUID
    name TEXT NOT NULL,              -- 记忆体名称
    bound_user_id TEXT,              -- 绑定用户ID（NULL表示未绑定）
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- LLM 配置表
CREATE TABLE llm_configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    provider TEXT NOT NULL,          -- deepseek, openai, qianwen 等
    api_key TEXT NOT NULL,           -- 加密存储
    model TEXT NOT NULL,             -- 模型名称
    temperature REAL DEFAULT 0.7,
    max_tokens INTEGER,
    created_at TIMESTAMP
);
```

| 类型 | 提供商 | 模型 |

- API Key 存储在记忆体数据库中（加密）
- 不在 `settings.json` 中存储敏感信息
- 支持温度、最大 token 等参数配置

## 五、账号系统

### 功能

| 功能 | 说明 |
|------|------|
| **用户注册** | 邮箱 + 密码 + 昵称 |
| **用户登录** | 邮箱 + 密码登录 |
| **密钥派生** | PBKDF2 派生加密密钥 |
| **记忆体绑定** | 登录后可绑定记忆体到账号 |

### 安全特性

- 密码使用 PBKDF2 + salt 哈希存储
- API Key 使用 AES-256 加密
- 加密密钥从用户密码派生，不明文存储
